# 测试工作流技能

## 概述

此技能演示了一个完整的多步骤工作流，用于测试技能执行流程是否正确。它包含数据收集、处理、验证和报告生成四个阶段，展示了技能如何协调多个工具完成复杂任务。

## 使用场景

在以下情况下使用此技能：
- 测试技能系统的工作流程是否正常
- 验证多步骤任务的执行流程
- 演示技能如何组织和协调多个工具调用
- 检查技能返回的执行步骤是否能被正确解析和执行

## 工作流决策树

```
开始测试工作流
    │
    ▼
步骤1: 数据收集 ──→ 使用工具收集测试数据
    │
    ▼
步骤2: 数据处理 ──→ 处理和分析收集的数据
    │
    ▼
步骤3: 结果验证 ──→ 验证处理结果的正确性
    │
    ▼
步骤4: 报告生成 ──→ 生成测试报告
    │
    ▼
   完成
```

## 执行步骤

### 步骤1: 数据收集

**目标**: 收集测试所需的基础数据

**操作**:
1. 读取配置文件（如有）
2. 收集环境信息
3. 记录当前工作目录和系统状态

**预期工具调用**:
- `Read` - 读取配置文件
- `LS` - 列出目录内容
- `Grep` - 搜索特定信息

### 步骤2: 数据处理

**目标**: 处理收集到的数据

**操作**:
1. 分析收集的数据
2. 执行数据转换
3. 计算统计信息

**预期工具调用**:
- `RunCommand` - 执行数据处理脚本
- `Write` - 写入中间结果

### 步骤3: 结果验证

**目标**: 验证处理结果

**操作**:
1. 检查输出文件是否存在
2. 验证数据格式
3. 确认结果符合预期

**预期工具调用**:
- `Read` - 读取处理结果
- `Glob` - 查找输出文件

### 步骤4: 报告生成

**目标**: 生成测试执行报告

**操作**:
1. 汇总各步骤执行结果
2. 生成Markdown格式的报告
3. 保存报告到指定位置

**预期工具调用**:
- `Write` - 写入最终报告

## 示例

### 示例1: 完整工作流测试

**用户输入**: "运行测试工作流，验证技能系统"

**执行流程**:
1. 检测到此技能被触发
2. 返回执行步骤给大模型
3. 大模型按顺序调用工具：
   - `LS` 列出当前目录
   - `Read` 读取测试配置
   - `RunCommand` 执行测试脚本
   - `Write` 生成测试报告
4. 返回最终结果给用户

### 示例2: 单步骤测试

**用户输入**: "只执行数据收集步骤"

**执行流程**:
1. 识别只需要执行步骤1
2. 调用 `LS` 和 `Read` 收集信息
3. 返回收集结果

## 脚本资源

此技能包含以下脚本：

### scripts/collect_data.py
数据收集脚本，用于收集环境信息和测试数据。

### scripts/process_data.py
数据处理脚本，用于处理和分析收集的数据。

### scripts/generate_report.py
报告生成脚本，用于生成格式化的测试报告。

## 预期行为验证

使用此技能可以验证以下功能：

1. **技能检测**: 验证技能是否能被正确识别和触发
2. **执行步骤返回**: 验证技能返回的执行步骤格式正确
3. **工具协调**: 验证多个工具能否按顺序协调执行
4. **结果汇总**: 验证多步骤结果能否正确汇总
