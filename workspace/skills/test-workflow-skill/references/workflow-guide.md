# 测试工作流执行指南

## 概述

本文档详细说明测试工作流技能的执行流程和最佳实践。

## 工作流阶段详解

### 阶段1: 数据收集

#### 目标
收集执行测试所需的所有基础数据和环境信息。

#### 关键活动
1. **环境检测**
   - 操作系统类型和版本
   - Python版本
   - 当前工作目录
   - 系统时间

2. **目录扫描**
   - 列出目标目录的内容
   - 统计文件和目录数量
   - 收集文件元数据（大小、修改时间）

3. **配置读取**
   - 查找配置文件
   - 解析配置内容
   - 验证配置有效性

#### 预期输出
```json
{
  "collection_time": "2026-02-20T13:00:00",
  "environment": {
    "platform": "Windows",
    "python_version": "3.10.0"
  },
  "directory": {
    "file_count": 10,
    "directory_count": 5
  },
  "config": {
    "config_found": true
  }
}
```

### 阶段2: 数据处理

#### 目标
处理和分析收集到的原始数据。

#### 关键活动
1. **数据清洗**
   - 移除无效数据
   - 标准化数据格式
   - 处理缺失值

2. **数据分析**
   - 计算统计指标
   - 识别数据模式
   - 生成分析摘要

3. **数据转换**
   - 转换为目标格式
   - 生成可视化数据
   - 准备报告数据

#### 分析方法

**摘要分析 (summary)**
- 总项目数
- 成功/失败/跳过数量
- 通过率
- 总耗时

**详细分析 (detailed)**
- 包含摘要分析的所有内容
- 每个项目的详细结果
- 性能评估
- 异常检测

### 阶段3: 结果验证

#### 目标
验证数据处理结果的正确性和完整性。

#### 验证项
1. **文件验证**
   - 输出文件是否存在
   - 文件大小是否合理
   - 文件格式是否正确

2. **数据验证**
   - 数据完整性检查
   - 数值范围验证
   - 格式一致性检查

3. **逻辑验证**
   - 计算结果合理性
   - 业务逻辑正确性
   - 关联数据一致性

### 阶段4: 报告生成

#### 目标
生成格式化的测试执行报告。

#### 报告结构
1. **执行摘要**
   - 执行时间
   - 步骤统计
   - 成功率
   - 总耗时

2. **详细结果**
   - 每个步骤的执行状态
   - 耗时分析
   - 输出内容

3. **结论和建议**
   - 总体评价
   - 改进建议
   - 后续行动

## 执行模式

### 完整工作流模式
执行所有四个阶段，适用于全面测试。

```
数据收集 → 数据处理 → 结果验证 → 报告生成
```

### 单阶段模式
只执行指定阶段，适用于针对性测试。

```
只执行: 数据收集
```

### 跳过模式
跳过某些阶段，适用于快速测试。

```
数据收集 → [跳过数据处理] → 结果验证 → 报告生成
```

## 最佳实践

### 1. 数据收集阶段
- 明确指定目标目录
- 检查文件权限
- 记录收集时间戳

### 2. 数据处理阶段
- 选择合适的分析类型
- 保存中间结果
- 处理异常情况

### 3. 结果验证阶段
- 设置明确的验证标准
- 记录验证日志
- 标记验证失败项

### 4. 报告生成阶段
- 选择合适的输出格式
- 包含必要的元数据
- 保存到指定位置

## 故障排除

### 常见问题

**数据收集失败**
- 检查目录权限
- 确认路径正确
- 查看错误日志

**数据处理错误**
- 验证输入数据格式
- 检查分析参数
- 查看堆栈跟踪

**报告生成失败**
- 确认输出目录可写
- 检查磁盘空间
- 验证模板完整性

## 扩展指南

2. 添加新的输出格式支持
